# SageMaster統合運用ダッシュボード 実装仕様書（確定版）

## プロジェクト概要
株式会社LEVEL1の社長個人資産運用（SageMaster経由）の実績集計・分析・将来予測システム。

- **対象1**：MEXC取引所（現物／AIグリッド運用）
- **対象2**：BigBoss取引所（FX／MelodyTrade - TP Ladderコピートレード）
- **目的**：
  1. コピートレード配信者（MelodyTrade）の実力を客観評価
  2. 月額$149のSageMasterサブスクが損益分岐を超えているか判定
  3. 将来資産シミュレーションで戦略判断を支援
  4. FXで増やした資金を現物AIグリッドへ移管する資産形成戦略を可視化
- **設計原則**：月額課金回避（GAS + 無料枠のみ）

---

## 2. スプシ構成（9シート）

### ① 📊 統合ダッシュボード
- 年月を横軸、KPIを縦軸
- **単一列構成**：過去=実績（緑）/当月=最新値（黄）/未来=予測（水色）
- GASが日付判定して自動色分け
- 統合指標：総資産（¥）、前月比、月次利益、累積経費、純利益

### ② 📈 現物_月次（MEXC）
- 通貨別の元本・現在値・利回り
- 対象：USDT, TIA, RENDER, LINK, ALGO, CAKE, ENA, PENDLE（今後追加可）
- 手数料控除後の純利益を算出
- 月利8%を予測値として展開（経費控除後の純利回り算出）

### ③ 📉 FX_月次（BigBoss / MelodyTrade）
- Balance（真正資産）とEquity（含みクレジット）を分離表示
- 月次KPI：総トレード数、勝率、PF、RR比、期待値、最大DD
- 平均ロット、最大ロット、証拠金維持率の推移
- **クレジットボーナス$3,128は出金不可のため「参考値」として別枠管理**

### ④ 🎯 目標vs実績対比
- 各指標の目標値・実績値・達成率を横並び
- 🟢🟡🔴の条件付き書式で一目判定
- 目標値（初期設定）：

| FX 月次利回り | ≥ +5.0% | 月 |

### ⑤ 👤 配信者スコアカード（MelodyTrade）

## 3. MT4 HTMLパース仕様

### 入力
- ファイル名：`DetailedStatement.htm`（MT4の詳細レポート保存機能）
- 保存先：Google Drive「SageMaster/FX/MT4_Reports/」フォルダ
- 運用：社長が週1回（月曜朝）MT4から保存→フォルダにアップロード

### パース対象データ

**取引明細（Closed Transactions）**
| カラム | HTMLソース | 用途 |
|---|---|---|
| Ticket | チケット番号 | 重複排除キー |
| Open Time | 建玉時刻 | 時間帯分析 |
| Type | buy/sell/balance/credit | 入出金/取引区分 |
| Size | ロット | リスク計算 |
| Item | 銘柄（gold_usd等） | 銘柄別集計 |
| Price / Close Price | 建値/決済価格 | pips計算 |
| S/L, T/P | 損切/利確ライン | リスク分析 |
| Commission / Swap / Profit | 損益要素 | 純損益算出 |

**サマリー**
- Deposit / Credit / Balance / Equity / Free Margin
- Closed P/L, Gross Profit, Gross Loss, **Profit Factor**, Expected Payoff
- Absolute DD, Maximal DD, Relative DD
- Total Trades, Win Rate, 平均利益/損失、連勝連敗

### パーサー擬似コード
```javascript
function parseMT4HTML(htmlBlob) {
  const html = htmlBlob.getDataAsString();
  const doc = XmlService.parse(cleanHTML(html));
  
  // 1. Closed Transactionsテーブル抽出
  const trades = extractTradeRows(doc);
  
  // 2. Deposit/Credit/Balance/Equity抽出
  const summary = extractSummary(doc);
  
  // 3. Detailed統計値抽出（PF, DD等）
  const stats = extractDetailedStats(doc);
  
  return { trades, summary, stats };
}
```

## 6. ロット最適化計算ロジック

### 安全ロット計算式
```
最大ロット = (口座残高 × リスク%) / (SL幅pips × pip価値)
```

### XAUUSD（ゴールド）の場合
- pip価値：1ロット = $10/pip
- SL平均：6pips（HTMLデータより）

| リスク% | 最大ロット（残高$6,220時） | 判定 |
|---|---|---|
| 1% | 0.10 | 超安全 |
| 2% | 0.20 | 安全 |
| 3% | 0.31 | 標準 |
| 5% | 0.52 | 現状 |
| 7% | 0.72 | 攻め（PF>1.3時） |
| 10% | 1.00 | 危険 |

### 段階的ロット引き上げロジック
```
if (直近30日 PF > 1.5) → リスク7%解禁
if (直近60日 PF > 1.5) → リスク10%検討
if (直近30日 PF < 1.0) → リスク2%に縮小
```
