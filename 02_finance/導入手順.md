# MF会計連携 PLレポート自動化システム 導入手順

## 概要

マネーフォワード クラウド会計（MF会計）のAPIを使って、
部門別・全体のPL月次推移表をGoogle スプレッドシートに自動生成するシステムです。

### 出力されるシート（事業年度ごとに自動作成）

| シート名例 | 内容 |
|---|---|
| `第8期_PL_共通` | 共通部門 月次PL（3〜2月）|
| `第8期_PL_物販` | 物販部門 月次PL |
| `第8期_PL_ブランド` | ブランド部門 月次PL |
| `第8期_PL_民泊` | 民泊部門 月次PL |
| `第8期_PL_全体` | 全部門統合 月次PL |
| `第8期_推移_部門別` | 部門別横断 推移表 |
| `第8期_推移_全体` | 全体 月次推移＋売上比 |

### 自動更新スケジュール
- **毎月1日 AM6:00** に当期データを自動更新

---

## STEP 1: MF会計でAPIアプリを登録

1. MF会計にログイン
2. **右上のアカウントメニュー → API連携** を開く
3. **「アプリを登録する」** をクリック
4. 以下を入力してアプリを作成：
   - アプリ名: `PLレポート自動化`
   - リダイレクトURI: `urn:ietf:wg:oauth:2.0:oob`
   - スコープ: `mf_clerk`
5. 表示された **クライアントID** と **クライアントシークレット** をコピーしておく

---

## STEP 2: Google スプレッドシートを準備

1. Google ドライブで新しいスプレッドシートを作成
2. URLからスプレッドシートIDをコピー
   - 例: `https://docs.google.com/spreadsheets/d/`**`1ABC...xyz`**`/edit`
   - 太字部分がスプレッドシートID

---

## STEP 3: Google Apps Script プロジェクトを作成・デプロイ

### 方法A: clasp（推奨・自動デプロイ）

```bash
# 前提: Node.js インストール済み
cd 02_finance
npm install

# Googleアカウントでログイン
npx clasp login

# 新規GASプロジェクトを作成（スプレッドシートに紐づける）
npx clasp create --type sheets --title "PLレポート自動化" --rootDir gas

# 作成されたスクリプトIDを gas/.clasp.json に記録（自動記録されます）

# GASにデプロイ
npm run deploy
```

### 方法B: 手動でApps Scriptに貼り付け

1. スプレッドシートを開く
2. **拡張機能 → Apps Script** を開く
3. `gas/` フォルダ内の各 `.gs` ファイルの内容を貼り付け
   （ファイルは別ファイルとして追加: 「+」ボタン → スクリプト）
4. `appsscript.json` の内容を左サイドバーの「プロジェクトの設定 → appsscript.json を表示する」から貼り付け

---

## STEP 4: Config.gs を設定する

GASエディタで `Config.gs` を開き、`★要設定` の箇所を入力します。

```javascript
MF_API: {
  CLIENT_ID:     'xxxxxxxxxxxxxxxx',  // ← STEP 1でコピーしたID
  CLIENT_SECRET: 'xxxxxxxxxxxxxxxx',  // ← STEP 1でコピーしたシークレット
},

SPREADSHEET_ID: '1ABC...xyz',  // ← STEP 2でコピーしたID
```

> ⚠️ `DEPARTMENTS` の部門IDはSTEP 6で設定します

---

## STEP 5: MF会計の認証を行う

1. スプレッドシートを開く（**一度リロード**するとメニューに「📊 財務レポート」が追加される）
2. **📊 財務レポート → 🔑 MF会計 認証を開始** を実行
3. 表示されたURLをクリックしてMF会計にログイン
4. 「許可する」をクリック → 表示された**認証コード**をコピー
5. **📊 財務レポート → 🔑 認証コードを入力** を実行 → コードを貼り付け
6. 「✅ 認証が完了しました」が表示されればOK

---

## STEP 6: 部門IDを設定する

1. **📊 財務レポート → 🏢 部門一覧を出力** を実行
2. 出力された `_部門確認` シートで各部門のIDを確認
3. `Config.gs` の `DEPARTMENTS` に各部門のIDを設定：

```javascript
DEPARTMENTS: [
  { id: 'abcd1234',  name: '共通',   shortName: '共通'   },
  { id: 'efgh5678',  name: '物販',   shortName: '物販'   },
  { id: 'ijkl9012',  name: 'ブランド', shortName: 'ブランド' },
  { id: 'mnop3456',  name: '民泊',   shortName: '民泊'   },
],
```

4. 変更後、GASエディタから **clasp push** または手動で更新して保存

---

## STEP 7: 勘定科目マッピングを確認・調整する（必要な場合）

1. **📊 財務レポート → 📋 勘定科目一覧を出力** を実行
2. 出力された `_勘定科目確認` シートでMF会計の科目名を確認
3. `Config.gs` の `PL_STRUCTURE` 内 `accountNames` を実際の科目名に合わせる

   例: MF会計で「売上高」が「商品売上」という名前の場合:
   ```javascript
   { category: 'revenue', label: '売上（国内）',
     accountNames: ['商品売上', '売上高'], ... }  // ← 実名を追加
   ```

---

## STEP 8: 動作確認

1. **📊 財務レポート → 🔄 当期PLを更新** を実行
2. 約2〜3分後、シートが生成されたことを確認
3. 数値が正しいかMF会計のレポートと照合

---

## STEP 9: 月次トリガーを設定する

1. **📊 財務レポート → ⚙️ 月次トリガーを設定** を実行
2. 「✅ トリガー設定完了」が表示されればOK
3. 毎月1日 AM6:00 に自動更新が実行されます

---

## 運用ガイド

### 毎月の確認事項
- 月次更新後（毎月1日以降）、シートの数値をMF会計のレポートと照合
- 差異がある場合は `_エラーログ` シートでエラーを確認

### 新しい事業年度（3月）の開始時
- 何もしなくても自動で新しい期のシートが生成されます
- 翌年度のシート名例: `第9期_PL_物販` など

### 勘定科目を追加したとき
- MF会計で新しい勘定科目を追加した場合は `Config.gs` の `accountNames` に追加

### トラブルシューティング

| 症状 | 対処法 |
|---|---|
| 「トークンが見つかりません」エラー | STEP 5の認証をやり直す |
| 数値が0になる | `_勘定科目確認` シートで科目名を確認し `accountNames` を修正 |
| 「事業所が見つかりません」エラー | Config.gs の CLIENT_ID / SECRET を再確認 |
| シートが作成されない | SPREADSHEET_ID が正しいか確認 |

## ファイル構成

```
02_finance/
├── gas/
│   ├── Config.gs        # ★設定ファイル（初期設定で編集必須）
│   ├── Main.gs          # エントリーポイント・メニュー定義
│   ├── Setup.gs         # 認証・トリガー設定
│   ├── MFApiClient.gs   # MF会計 API通信クライアント
│   ├── PLFormatter.gs   # PL集計・フォーマットロジック
│   ├── SheetManager.gs  # スプレッドシート書き込み
│   ├── appsscript.json  # GASマニフェスト
│   └── .clasp.json      # clasp設定（デプロイ用）
├── package.json         # npm設定（clasp用）
└── 導入手順.md           # このファイル
```
