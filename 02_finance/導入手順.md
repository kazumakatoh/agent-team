# MF会計連携 PLレポート自動化システム 導入手順

## 概要

マネーフォワード クラウド会計（MF会計）のAPIを使い、
部門別・全体のPL月次推移表を Google スプレッドシートに自動生成するシステムです。

### 出力シート（事業年度ごとに自動作成）

| シート名 | 内容 |
|---|---|
| `第8期_PL_共通` / `物販` / `ブランド` / `民泊` | 部門別 月次PL（3〜2月）+ 各月売上比 |
| `第8期_PL_全体` | 全部門統合 月次PL |
| `第8期_推移_部門別` | 部門横断 月次推移表 |
| `第8期_推移_全体` | 全体 月次推移 + 売上比 |

**毎日 AM6:00 に当月以前のデータを自動更新**（未来月の手入力予測値は保持）

---

## ⚠️ 重要：作業の順番を守ってください

```
STEP 1: Googleスプレッドシートを作成
STEP 2: GASにコードをデプロイ → Web App URLを取得
STEP 3: MFアプリポータルでアプリ登録（Web App URLを使用）
STEP 4: Config.gs に設定値を入力
STEP 5: MF会計で認証
STEP 6: 部門IDを確認・設定
STEP 7: 動作確認 → 自動トリガー設定
```

---

## STEP 1: Google スプレッドシートを作成

1. Google ドライブで **新しいスプレッドシート** を作成
2. URLからスプレッドシートIDをコピーしてメモ
   - 例: `https://docs.google.com/spreadsheets/d/`**`1ABC...xyz`**`/edit`
   - 太字部分（`1ABC...xyz`）がスプレッドシートID

---

## STEP 2: GASにコードをデプロイする

### 方法A: claspを使う（推奨）

```bash
# 前提: Node.js インストール済み
cd 02_finance
npm install

# Googleアカウントでログイン
npx clasp login

# スプレッドシートに紐づく新規GASプロジェクトを作成
npx clasp create --type sheets --title "PLレポート自動化" --rootDir gas

# GASにコードをプッシュ
npm run deploy
```

### 方法B: GASエディタに手動貼り付け

1. STEP 1で作成したスプレッドシートを開く
2. **拡張機能 → Apps Script** を開く
3. `gas/` フォルダの各 `.gs` ファイルを追加・貼り付ける
   - 「+」ボタン → スクリプト を選択してファイルを追加
   - ファイル名と内容を合わせて貼り付ける（Config.gs, Main.gs, Setup.gs, MFApiClient.gs, PLFormatter.gs, SheetManager.gs）
4. `appsscript.json` の内容を反映する
   - GASエディタ左サイドバー「プロジェクトの設定」→「appsscript.json を表示する」をオン
   - 表示された `appsscript.json` の内容を `gas/appsscript.json` の内容に置き換える

### Web App としてデプロイする

**これが最重要ステップです。** OAuth2のコールバックを受け取るために必要です。

1. GASエディタ右上の **「デプロイ」→「新しいデプロイ」** をクリック
2. 種類の選択で **「ウェブアプリ」** を選択
3. 設定を入力：
   - 説明: `PLレポート自動化 OAuth`
   - 次のユーザーとして実行: **「自分」**
   - アクセスできるユーザー: **「全員」**
4. **「デプロイ」** をクリック
5. 表示された **ウェブアプリのURL** をコピーしてメモ
   - 例: `https://script.google.com/macros/s/AKfycbXXXXXX.../exec`

> 💡 デプロイ後、スプレッドシートを開いて「📊 財務レポート」メニューが表示されれば成功。
> メニューの「📋 Web App URL を確認」でもURLを確認できます。

---

## STEP 3: MFアプリポータルでアプリを登録する

1. **app-portal.moneyforward.com/apps/new/** を開く（ログイン済みの状態で）
2. フォームに入力する：
   - **アプリ名称**: `PL/BS自動集計`（任意）
   - **リダイレクトURI**: STEP 2でコピーした Web App URL を貼り付け
     - 例: `https://script.google.com/macros/s/AKfycbXXXXXX.../exec`
   - **クライアント認証方式**: `CLIENT_SECRET_BASIC`（デフォルトのまま）
3. 「マネーフォワード クラウド API利用規約」を確認してチェック
4. **「登録」** をクリック
5. 表示された **クライアントID** と **クライアントシークレット** をコピーしてメモ

> ⚠️ クライアントシークレットはこの画面でしか確認できません。必ずメモしてください。

---

## STEP 4: Config.gs に設定値を入力する

GASエディタで `Config.gs` を開き、以下を設定します。

```javascript
MF_API: {
  CLIENT_ID:    'xxxxxxxxxxxxxxxx',  // ← STEP 3でコピーしたクライアントID
  CLIENT_SECRET:'xxxxxxxxxxxxxxxx',  // ← STEP 3でコピーしたシークレット
  REDIRECT_URI: 'https://script.google.com/macros/s/AKfycb.../exec', // ← STEP 2のURL
},

SPREADSHEET_ID: '1ABC...xyz',  // ← STEP 1でコピーしたID
```

設定後、**clasp push** または GASエディタで **Ctrl+S（保存）** してください。

---

## STEP 5: MF会計で認証する

1. スプレッドシートを開く（**一度リロード** でメニューが再表示される）
2. **📊 財務レポート → 🔑 MF会計 認証を開始** をクリック
3. ダイアログの「MF会計で認証する」をクリック
4. MF会計にログイン → 「許可する」をクリック
5. ブラウザに **「✅ MF会計 認証が完了しました」** と表示されれば成功
6. ダイアログを閉じてスプレッドシートに戻る

---

## STEP 6: 部門IDを確認・設定する

1. **📊 財務レポート → 🏢 部門一覧を出力** を実行
2. 表示された `_部門確認` シートで各部門のIDを確認
3. `Config.gs` の `DEPARTMENTS` に各部門のIDを設定：

```javascript
DEPARTMENTS: [
  { id: 'xxxxxxxx',  name: '共通',    shortName: '共通'    },
  { id: 'yyyyyyyy',  name: '物販',    shortName: '物販'    },
  { id: 'zzzzzzzz',  name: 'ブランド', shortName: 'ブランド' },
  { id: 'wwwwwwww',  name: '民泊',    shortName: '民泊'    },
],
```

4. 保存後、**clasp push** または Ctrl+S

> 💡 部門名がMF会計の設定と異なる場合は `name` の値を合わせてください。

---

## STEP 7: 勘定科目マッピングを確認する

1. **📊 財務レポート → 🔬 API生データを出力（初回診断用）** を実行
2. `_API生データ確認` シートで実際のJSONフィールド名を確認
3. **📊 財務レポート → 📋 勘定科目一覧を出力** を実行
4. `_勘定科目確認` シートのMF会計での科目名を確認
5. `Config.gs` の `PL_STRUCTURE` 内 `accountNames` を実際の科目名に合わせて修正

   例: MF会計で「売上高」が「商品売上」という名前の場合：
   ```javascript
   { category: 'revenue', label: '売上（国内）',
     accountNames: ['商品売上', '売上高'], ... }
   ```

---

## STEP 8: 動作確認 → 自動トリガー設定

1. **📊 財務レポート → 🔄 当期PLを更新（過去・当月のみ）** を実行
2. 約2〜3分後にシートが生成されることを確認
3. 数値をMF会計のレポートと照合
4. 問題なければ **📊 財務レポート → ⚙️ 日次トリガーを設定** を実行
5. **「✅ トリガー設定完了」** が表示されれば完了

**毎日 AM6:00 に自動更新が実行されます。**

---

## 運用ガイド

### 予測値の手入力（未来月）
- 未来月の金額セルに手動で予測値を入力できます
- 自動更新は**当月以前のみ**上書きするため、予測値は保持されます
- 売上比（比率列）は数式なので、手入力した予測値も自動で比率が計算されます

### 新しい事業年度（3月）の開始時
- 何もしなくても自動で新しい期のシートが生成されます
- 新年度のシートは3月1日のAM6:00更新で初回作成されます

### コードを更新した後
- `clasp push` または GASエディタで保存後
- 「デプロイ」→「デプロイを管理」→「✏️（編集）」→「バージョン: 新バージョン」→「デプロイ」

### トラブルシューティング

| 症状 | 対処法 |
|---|---|
| 認証画面でエラー | REDIRECT_URI がMFポータルと Config.gs で一致しているか確認 |
| 「トークンが見つかりません」 | STEP 5の認証をやり直す |
| 数値がすべて0 | `_API生データ確認` でフィールド名を確認し PLFormatter.gs を修正 |
| 部門ごとにデータが分かれない | DEPARTMENTS の部門IDが正しいか確認 |
| シートが作成されない | SPREADSHEET_ID が正しいか確認 |

## ファイル構成

```
02_finance/
├── gas/
│   ├── Config.gs        # ★設定ファイル（初期設定で必ず編集）
│   ├── Main.gs          # エントリーポイント・メニュー定義
│   ├── Setup.gs         # OAuth2コールバック・トリガー設定
│   ├── MFApiClient.gs   # MF会計 APIクライアント
│   ├── PLFormatter.gs   # PL集計・フォーマットロジック
│   ├── SheetManager.gs  # スプレッドシート書き込み
│   ├── appsscript.json  # GASマニフェスト（Web App設定含む）
│   └── .clasp.json      # clasp設定（デプロイ用）
├── package.json
└── 導入手順.md（このファイル）
```
