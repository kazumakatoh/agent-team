> **このドキュメントの位置付け**

## 1. 目的（更新）

健康診断・各種検査・日々のバイタルを蓄積し、

- **点ではなく線・面で見る**：単発の数値ではなく、年次推移・項目間の相関で俯瞰
- **正しく認識する**：基準値・年齢相応・自分自身の経時変化の3軸で評価
- **死因TOP5のリスクを下げる**：特に**がん・心疾患・脳血管疾患**の3大領域を重点管理
- **予防検査の漏れを防ぐ**：必要な検査が実施されているかを継続管理
- **健康寿命を延ばす**：早期発見＋実行可能な改善アクション提案

## 2. 戦略的フォーカス（リスクモデル）

### 2.1 日本人の死因TOP5と本システムの重点領域

| 順位 | 死因 | 構成割合 | 本システムの対応 |
|---|---|---|---|
| 1 | 悪性新生物（がん） | 24.0% | **重点**：男性部位別TOP5＋予防検査管理 |
| 2 | 心疾患 | 14.8% | **重点**：高血圧・脂質・糖代謝・心電図 |
| 3 | 老衰 | 12.1% | スコープ外（運動・栄養・睡眠で間接対応） |
| 4 | 脳血管疾患 | 6.6% | **重点**：高血圧・頸動脈・心房細動 |
| 5 | 不慮の事故 | 2.8% | スコープ外 |

→ **重点3領域（がん・心疾患・脳血管疾患）で死因の45.4%をカバー**

### 2.2 男性のがん部位別TOP5と予防検査プロトコル

| 順位 | 部位 | シェア | 主要予防検査 | 推奨頻度 | 優先確認事項 |
|---|---|---|---|---|---|
| 1 | 肺 | 23.9% | 低線量CT、胸部X線 | 年1回（喫煙歴有なら半年） | 喫煙歴・受動喫煙環境、CT実施有無 |
| 2 | 大腸 | 12.6% | 便潜血検査、大腸内視鏡 | 便潜血:年1、内視鏡:5年に1回 | 内視鏡受診歴 |
| 3 | 胃 | 11.4% | **ピロリ菌検査**、胃内視鏡 | ピロリ:1回（陰性なら以降不要）、内視鏡:1〜2年 | **ピロリ菌結果（陰性確定なら大幅リスク減）** |
| 4 | 膵臓 | 9.0% | 腹部エコー、CA19-9（腫瘍マーカー）、糖尿病管理 | 年1回 | 糖尿病既往、家族歴 |
| 5 | 肝臓 | 6.9% | 肝炎ウイルス検査（HBV/HCV）、腹部エコー、AFP | ウイルス:1回、エコー:年1 | ウイルス検査結果 |

### 2.3 心疾患・脳血管疾患の予防検査

| 領域 | 主要検査 | 推奨頻度 |
|---|---|---|
| 心疾患 | 心電図、心エコー、頸動脈エコー、冠動脈CT（リスク高い場合） | 年1〜数年に1回 |
| 脳血管疾患 | 頸動脈エコー、脳MRI/MRA（脳ドック） | 数年に1回 |

→ これらをすべて `exam_schedule.gsheet` で管理し、漏れをGmail通知。

## 4. 完成イメージ（運用ストーリー）

### A. 健診/検査結果が出たとき
1. 社長がPDFを Drive `12_加藤家/01_健康/01_inbox/` に保存
2. GASが新規PDFを検知し、Claude APIで項目抽出
3. 項目辞書で正規化されたデータが `master_health` に追記
4. 辞書未登録の新項目があれば `pending_dictionary` に候補蓄積、Gmailで社長に確認依頼
5. Claudeが**健診サマリーレポート**を生成、Gmail＋ `analysis/` に保存
6. 社長が確認、誤抽出は手動修正

### B. 月次バイタルレビュー（毎月初）
1. GASがFitbit/Google Health APIから前月のバイタルデータをプル
2. `master_vitals` に書き込み
3. Claudeが**月次バイタルレビュー**を生成
4. **秘書エージェントの朝レポートに統合**して通知

### C. 四半期レビュー（年4回）
1. Claudeが直近健診×3ヶ月バイタル×検査スケジュール横断分析
2. **3大領域のリスクスコア更新**
3. 改善アクション進捗評価＋次四半期KPI更新
4. Gmail送信

### D. 予防検査スケジュール管理（毎月チェック）
1. GASが `exam_schedule` をスキャン
2. **次回予定日を過ぎている検査**があればリスト化
3. 例：「肺がん予防の低線量CTが2年未受診」「ピロリ菌検査未実施」
4. **Gmail通知＋秘書朝レポートに表示**
5. 社長が予約 → 受診 → PDF登録 → 自動的に `last_done` 更新

### E. アラート（随時）
- トリガー：基準値の重大逸脱、3ヶ月連続悪化、急変、検査漏れ
- 配信：Gmail即時送信＋朝レポート

## 6. API選択：Fitbit/Google Fit廃止対応

### 6.1 調査結果サマリー

**重要：両方とも2026年中に廃止される過渡期**

| API | 状況 | 期限 |
|---|---|---|
| **Fitbit Web API** | 廃止予定 | 2026年9月停止、データダウンロードは2026年7月15日まで |
| **Google Fit REST API** | 廃止予定 | 2026年末で終了、新規開発者登録は2024年5月で停止済 |
| **Google Health API**（新） | **後継・推奨** | 2026年現在の正式パス |
| **Health Connect**（Android） | モバイル中心の代替 | Android端末必要 |

### 6.2 推奨パス：Google Health API

- Fitbit/Google Fit双方の後継として設計された統一API
- Google OAuth 2.0で認証
- GASからのアクセスが可能（OAuth2ライブラリ経由）
- Fitbit利用者は **2026年9月までに移行必須**

### 6.3 移行スケジュール上の留意点
- 社長がFitbitを継続利用する場合：2026年9月までにGoogle Health API移行
- データダウンロード期限：2026年7月15日（Fitbit旧データの最終救済期限）
- → **本プロジェクトは最初からGoogle Health APIで構築**するのが合理的

### 6.4 アカウント情報（重要・機微）
- **Fitbit / Google Health API用Googleアカウント**：`kazuma.katoh.0406@gmail.com`（個人専用、会社用とは別）

## 7. データモデル

### 7.1 健診/検査マスター（`master_health`）
1行 = 1検査日 × 1項目（ロングフォーマット）

| 列名 | 例 | 備考 |
|---|---|---|
| exam_date | 2026-04-15 | 検査日 |
| facility | ○○クリニック | 受診医療機関 |
| category | 定期健診 | 定期健診 / 人間ドック / 専門検査 |
| specialty | 内科 | 内科/眼科/整形/泌尿器/その他 |
| item_code | hba1c | 辞書コード（正規化済み） |
| item_name_raw | HbA1c | PDF表記そのまま |
| value | 5.4 | 数値 |
| unit | % | 単位 |
| ref_low | 4.6 | 基準値下限 |
| ref_high | 6.2 | 基準値上限 |
| judgment | A | A/B/C/D 等 |
| risk_link | 糖代謝/心疾患 | 関連する3大領域リスク |
| source_pdf | 20260415_xxclinic.pdf | 出典PDFファイル名 |
| extracted_by | AI | AI / 手動 |
| confirmed | TRUE | 社長確認済みフラグ |
| note | | 任意メモ |

### 7.2 項目辞書（`item_dictionary`）

**ハイブリッド方式：基本辞書（30〜50項目）を初期投入＋実PDFから新項目を自動候補化**

**社長指定の必須項目（基本辞書に確実に含める）**：
- 脂質関連：総コレステロール / HDLコレステロール / LDLコレステロール / 中性脂肪（TG）/ Non-HDL / LH比
- 血液項目：白血球数 / 赤血球数 / ヘモグロビン / ヘマトクリット / 血小板 / MCV / MCH / MCHC
- 肺機能：肺活量 / %肺活量 / FEV1 / FEV1% / %FEV1（スパイロメトリー一式）
- 糖代謝：HbA1c / 空腹時血糖 / 随時血糖
- 肝機能：AST / ALT / γ-GTP / ALP / 総ビリルビン / 総蛋白 / アルブミン
- 腎機能：BUN / クレアチニン / eGFR / 尿酸
- 血圧・身体計測：収縮期/拡張期血圧 / 体重 / BMI / 腹囲
- ホルモン（テストステロン関連は別シート連動）：総テストステロン / 遊離テストステロン

| 列名 | 例 |
|---|---|
| item_code | hba1c |
| canonical_name | HbA1c（NGSP） |
| aliases | HbA1c, ヘモグロビンA1c, グリコヘモグロビン |
| category | 糖代謝 |
| risk_link | 糖尿病/心疾患 |
| unit_canonical | % |
| ref_low_default | 4.6 |
| ref_high_default | 6.2 |
| description | 過去1〜2ヶ月の平均血糖値の指標 |
| status | confirmed | confirmed / pending |

新項目検出時の運用：
1. PDF抽出で辞書未登録の項目を発見
2. `pending_dictionary` に候補追加（AIが項目名・カテゴリ・基準値を推測）
3. Gmail通知「○○件の新項目が見つかりました。確認してください」
4. 社長承認後、`item_dictionary` に正式登録

## 8. AI分析テンプレート

### 8.1 健診サマリー（年次）
- **冒頭固定文**：「本レポートは医師の診断ではなく、気づきの提供です」
- 受診情報
- **3大領域リスクスコア**（がん/心疾患/脳血管疾患それぞれを5段階評価）
- 項目別ステータス（基準値内 / 軽度逸脱 / 要注意 / 要受診）
- 前年比・5年推移
- **改善アクションプラン**（次回健診までのKPI付き）

### 8.2 月次バイタルレビュー
- 月平均と前月比、3ヶ月移動平均
- 異常値日（睡眠不足5日連続など）
- 健診懸念項目との関連推測
- 当月の重点アクション
- → 秘書朝レポートに統合

### 8.3 四半期総合分析
- 健診×バイタル×検査スケジュール横断
- 3大領域リスクスコア推移
- 改善アクションプラン進捗（達成度％）
- 次四半期KPI

### 8.4 予防検査スケジュールレビュー（毎月）★新規
- **未実施または期限超過の検査リスト**
- 例：「肺がん予防の低線量CTが2年未受診です。年1回の受診を推奨します」
- 「ピロリ菌検査が未実施です。胃がんリスクの最重要予防策です」
- データソース：`exam_protocols.md`（初期）→ `exam_schedule.gsheet`（フェーズ1B以降）
- → Gmail通知＋朝レポート

### 8.5 テストステロン専用レビュー（半年〜年1回）★新規
- 総テストステロン・遊離テストステロン・SHBG・DHEA-Sの推移グラフ
- 年齢相応の基準値との比較（男性更年期評価）
- 関連症状・QOLメモ欄との照合
- バイタル（睡眠・運動量）との相関分析
- テンプレート：`analysis_templates/testosterone_review.md`（フェーズ1Aで作成）

### 8.6 アラート
- 基準値の30%以上逸脱、3ヶ月連続悪化、急変、検査漏れ
- → Gmail即時送信
