# 日次ルーティン設計
*最終更新：2026年3月*
*場所：00_経営ダッシュボード*

---

## 🌅 毎朝の使い方

**9時以降に「おはよう」と話しかけるだけ。**
Claudeが以下のフォーマットで日次レポートを返す。

---

## 📋 日次レポート フォーマット

```
【日次レポート】YYYY年MM月DD日（曜日）

☀️ 今日の天気（東京）
　→ 天気・最高/最低気温・一言

📅 今日の予定
　→ 時間 / 内容（Googleカレンダーから取得）
　→ なければ「予定なし」

⚠️ 今週の重要期限
　→ 締切が近いものを最大3件

🆕 YADORIGIの今日のTODO
　→ 年間カレンダーから該当タスクを抜粋

💬 一言メモ
　→ 気になること・先回り提案（1行だけ）
```

---

## 📌 運用ルール

- **トリガー**：「おはよう」と話しかける
- **時間帯**：9時以降
- **長さ**：30秒で読める・箇条書き・結論ファースト
- **Amazon売上データ**：Gemini構築完了後に追加する（現在は項目なし）

---

## 🔮 Gemini構築後に追加する項目

```
📦 Amazon 昨日の実績（Geminiレポートより）
　→ 売上合計：¥XXX,XXX
　→ 注目商品：商品名（前日比 +X%）
　→ 要確認：商品名（TACOS XX% → 要対応）
```

---

## 💬 その他の使い方（日中）

「おはよう」以外でも、随時以下のように使える：

| 話しかけ方 | Claudeがやること |
|---|---|
| 「今週何があったっけ？」 | 週次サマリーを返す |
| 「〇〇さんにメール書いて」 | 下書きを作成（送信は一馬さんが確認） |
| 「YADORIGIの進捗どうだっけ？」 | 年間カレンダーから現状を整理して返す |
| 「今月の資金繰り確認して」 | 借入返済・支出予定を整理して返す |
| 「プライムデーまであと何日？」 | 期日と準備状況を確認して返す |

## 📋 AIへのメモ
- 日次レポートは**短く・箇条書き・30秒で読める**が絶対条件
- 長々と説明せず、アクションが必要なものだけ伝える
- Googleカレンダー・天気は毎回必ずチェックして返す
- Amazon売上はGemini連携完了まで項目を表示しない
