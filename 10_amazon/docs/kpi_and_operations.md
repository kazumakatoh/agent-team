## 通知設計

### 週次改善提案（Gmail）

- **送信タイミング**: 毎週月曜 8:00
- **宛先**: 社長個人メールアドレスのみ
- **生成方法**: GAS → Claude API (claude-sonnet-4-6) → Gmail送信
- **履歴保管**: Gmail受信箱のみ（スプレッドシートには残さない）

**提案内容**:

```
【収益改善提案】
・粗利率の低い商品とその原因
・仕入れ価格・販売価格の見直し候補
・撤退を検討すべき商品

【広告改善提案】
・ACOSが高すぎるキャンペーン・キーワード
・予算を増やすべき成果の出ているキャンペーン
・停止を検討すべきキーワード

【仕入れ・在庫提案】
・在庫切れリスクのある商品（発注タイミングの警告）
・売れていないのに在庫が多い商品
・今後注力すべき売れ筋商品
```

### 月次改善提案（Gmail）

- **送信タイミング**: 毎月1日 9:00
- **生成AIモデル**: Claude Opus 4.6 (claude-opus-4-6)（月次は深い分析のため上位モデル使用）
- **内容**: 月次サマリー + 上記提案 + 前月比分析

### 緊急アラート（LINE）

- **送信方法**: LINE Messaging API
- **宛先**: 社長個人のみ
- **トリガー条件**（例）:
  - 在庫切れ間近の商品（在庫数が直近7日平均売上の3日分以下）
  - 1日の広告費が異常値（直近7日平均の2倍超）
  - Settlement Report で大きな返金・調整金が発生

## GAS 自動実行スケジュール

| 実行時刻 | 処理内容 |
|---|---|
| 毎日 6:00 | SP-API データ取得 → シート②⑦ 更新 |
| 毎日 6:30 | シート④（商品別収益）再集計 |
| 毎週月曜 7:00 | シート⑤（週次レポート）・⑧（販売分析）更新 |
| 毎週月曜 8:00 | Claude API → Gmail で改善提案送信 |
| 毎月1日 8:00 | シート⑥（月次レポート）更新 |
| 毎月1日 9:00 | Claude API (Opus) → Gmail で月次改善提案送信 |
| Settlement到着時 | シート③（経費明細）取り込み・シート②の暫定→確定更新 |

### GAS実行時間制限への対応

GASは1回の実行が6分以内の制限あり。対策:

- SP-APIのレポート取得は**非同期**（createReport → ポーリング → download）のため、処理を分割
  1. トリガー1: レポート作成リクエスト送信 + reportId を ScriptProperties に保存
  2. トリガー2: 5分後にポーリング → ダウンロード → シート書き込み
- 商品数が多い場合はバッチ処理（50商品ずつ等）で分割

## 既存Excel資産の移行

### 移行対象

| 種別 | シート数 | 移行方法 |
|---|---|---|
| Weekly シート | 10 | 過去実績アーカイブに一括インポート |
| Monthly シート | 8 | 過去実績アーカイブに一括インポート |
| Amazon全体集計 | 12 | 過去実績アーカイブに一括インポート |

### 移行方針

- 商品マスター（ASIN・仕入単価・仕入先）→ 新シート①へ移行（最重要）
- 過去の売上・広告実績 → 別シート「過去実績アーカイブ」に格納
- 月次レポートの前年同月比で活用
- Excel共有後に移行マッピング設計を実施
