## レイヤー1: Amazon内粗利

Amazonから取得できるデータのみで計算。Settlement Reportの全経費カテゴリを差し引く。

```
売上（税込）
  − 売上原価（仕入履歴の有効単価 x 注文点数）
  − Amazon販売手数料（Referral Fee）
  − FBA配送代行手数料
  − FBA在庫保管料（月次・長期）
  − FBA関連追加料（返品処理・廃棄・ラベル貼付等）
  − 広告費（Sponsored Products / Brands / Display）
  − 返品・返金額
  − プロモーション費（クーポン等）
  − Amazon割引
  − Amazon月額プラン料金（4,900円/月）
  − 国内配送・代行料
  − 海外配送
  − 開発・消費税
  − その他Settlement Reportに含まれるすべての経費
  ＝ ①Amazon内粗利
```

※既存Excelの経費内訳（Amazon割引/販売・FBA手数料/国内配送・代行料/海外配送/開発・消費税）と整合

### 表示単位

| 単位 | Amazon内粗利 | 粗利率 |
|---|---|---|
| 事業全体 | 全商品合計 | 合計粗利/合計売上 |
| カテゴリ別 | カテゴリ内ASIN合計 | カテゴリ粗利/カテゴリ売上 |
| 商品(ASIN)別 | 個別商品 | 商品粗利/商品売上 |

## レイヤー2: 最終利益

Amazon内粗利から、Amazonレポートに含まれない外部経費を差し引く。

```
①Amazon内粗利（全商品合計）
  − Amence 業務委託費（44,000円/月）
  − M19 月額固定費（7,000円/月）
  − M19 成果報酬（月額広告費 x 6.5%）
  − 納品人件費（ママ友バイト実績）
  − その他ツール費用（あれば）
  ＝ ②最終利益
```

### 設計ルール

- 外部経費は**全商品合計に対してのみ**控除（商品別按分はしない）
- 入力: 社長が月次で「販促費マスター（M3）」に金額のみ手入力
- M19成果報酬: 広告費合計から自動計算
- M19運用開始: 2026年4月20日頃（以前はゼロ）

### 表示場所

- L1 事業ダッシュボード: 全体サマリーに最終利益を表示
- 週次Gmail通知: 最終利益を記載
- 月次Gmail通知: 最終利益 + 前月比 + 前年同月比

## ROI・資金回転率・入金の定義

既存シートに全カテゴリ共通で搭載されている指標。新システムでも同一ロジックで実装。

### 入金（計算値・実振込額ではない）

```
入金 = 売上 + Amazon割引(負値) - 仕入原価 - 販売FBA手数料 - 国内配送代行料 - 海外配送 - 関税消費税
※広告費は除外（カード決済のため資金回転率とは独立）
```

### ROI

```
ROI = 利益 / 仕入 × 100（%）
```

仕入投資に対してどれだけ利益が出ているか。

### 資金回転率

```
資金回転率 = 入金 / 仕入 × 100（%）
```

仕入に使った現金がどれだけ回収されているか。100%超 = 仕入資金を回収済み。

### 表示場所
- L1 事業ダッシュボード: 全体のROI・資金回転率
- L2 カテゴリ分析: カテゴリ別
- L3 商品分析: 商品別

## 仕入単価の管理方法

### 設計: 既存キャッシュフロー管理シートから自動連携

**社長の追加作業: ゼロ**。既存の月次棚卸し業務で入力している仕入単価をGASが自動読み取り。

### データフロー

```
【既存】キャッシュフロー管理シート（Google スプレッドシート）
  社長が毎月1日に入力（今まで通り）
  ・商品ヘッダ行にASIN行を1行追加（初回のみ）
  ・各月ブロック: 在庫/見込売上/棚卸原価/仕入単価/販売単価
       ↓
  GAS が毎月3日 AM6:00 に自動読み取り
  ・「仕入単価」行を検索
  ・E列以降の各商品の値を取得
  ・ASIN行で商品を識別
       ↓
【自動生成】M2 月次仕入単価テーブル
  | ASIN | 年月 | 仕入単価 |
  | B0XX | 2026-03 | ¥150 |
  | B0XX | 2026-04 | ¥160 |
       ↓
  D1 日次データの原価計算に使用
  ・その月に売れた商品 → その月の仕入単価で計算
```

### 既存シートへの変更（初回のみ）

キャッシュフロー管理シートの商品ヘッダ部分に **ASIN行を1行追加**:

```
行X: ASIN     | (空) | (空) | B0XXXXX | B0XXXXX | B0XXXXX | ...
行Y: 商品名   | (空) | (空) | 静電気10cm1枚 | SDカード128GB | BCI-381+380 | ...
```

→ これにより商品名ではなくASINで自動マッチングが可能になる

### 読み取りタイミング

- 社長の入力: 毎月1日
- GASの読み取り: **毎月3日 AM6:00**（余裕を持って）
- 読み取り結果: M2に自動追記

### 月中に仕入単価が変わった場合

- 月次精度で計算（月内の変動は翌月の入力で反映）
- Amazon Inventory APIで在庫急増（= 納品）を検知した場合、LINEで任意通知可能（強制ではない）

### 将来改良（Phase 5以降）

- FIFO（先入先出法）: 仕入数量 vs 販売数量を突合し、在庫ロット別の原価追跡
- 移動平均法: 在庫全体の平均原価を自動計算
