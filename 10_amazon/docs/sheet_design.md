# スプレッドシート設計

*v2.1 - 3層ダッシュボード構成に変更（2026-04-12）*

## 設計思想

社長が把握したいこと:
1. **Amazon事業全体**がうまくいっているか（俯瞰）
2. **カテゴリ単位**でどうか（中解像度）
3. **個別商品(ASIN)**の詳細（高解像度）

→ 「見るシート3枚 + データシート（裏側）」に分離

---

## シート一覧

### 見るシート（社長が普段開くのはここだけ）

| # | シート名 | 用途 | 更新頻度 |
|---|---|---|---|
| L1 | **事業ダッシュボード** | 全体サマリー + カテゴリ別 + 注意商品 | 毎日 6:30 |
| L2 | **カテゴリ分析** | ドロップダウンでカテゴリ選択→詳細表示 | 毎日 6:30 |
| L3 | **商品分析** | ドロップダウンでASIN選択→全時間軸表示 | 毎日 6:30 |

### データシート（裏側・GAS自動更新）

| # | シート名 | 内容 | 更新頻度 |
|---|---|---|---|
| D1 | 日次データ | 全商品x全日付の生データ | 毎日 6:00 |
| D2 | 経費明細 | Settlement Report確定値 | Settlement到着時 |
| D3 | 広告詳細 | キャンペーン/キーワード/検索用語 | 毎日 6:00 |

### マスターシート（手入力）

| # | シート名 | 内容 | 更新頻度 |
|---|---|---|---|
| M1 | 商品マスター | ASIN/商品名/カテゴリ/現在仕入単価 | 随時 |
| M2 | 仕入履歴 | 仕入単価の変遷ログ | 仕入時 |
| M3 | 販促費マスター | Amence/M19/人件費/その他 | 月次 |

## D1: 日次データ（GAS自動・暫定値）

1行 = 1商品(ASIN) x 1日。毎朝6:00に前日分を自動追記。
L1〜L3の全ダッシュボードの元データ。

| 列 | 内容 | 取得元 |
|---|---|---|
| 日付 | YYYY-MM-DD | 自動 |
| ASIN | 商品コード | Amazon |
| 商品名 | マスター参照 | M1 |
| カテゴリ | マスター参照 | M1 |
| 売上金額（円） | 税込売上 | SP-API |
| CV（注文件数） | 注文件数 | SP-API |
| 注文点数 | 販売個数 | SP-API |
| セッション数 | 訪問数 | SP-API (Traffic) |
| PV | ページビュー | SP-API (Traffic) |
| CTR (%) | セッション数/PV | 自動計算 |
| CVR (%) | CV/セッション数 | 自動計算 |
| Buy Box率 (%) | Buy Box獲得率 | SP-API (Traffic) |
| FBA手数料（円） | 配送・保管料（暫定） | SP-API |
| 返品数 | 個数 | SP-API |
| 返品額（円） | 返金額 | SP-API |
| 広告費（円） | その日の広告費 | Ads API |
| 広告売上（円） | 広告経由売上 | Ads API |
| IMP | インプレッション | Ads API |
| CT | クリック数 | Ads API |
| 仕入れ単価（円） | その時点の有効単価 | M2参照 |
| 仕入れ原価合計（円） | 仕入単価 x 注文点数 | 自動計算 |
| ステータス | 暫定 / 確定 | 自動 |

## M2: 月次仕入単価テーブル（GAS自動生成）

既存のキャッシュフロー管理シートからGASが毎月3日に自動読み取り。**手入力不要**。

| 列 | 内容 | 更新方法 |
|---|---|---|
| ASIN | 商品コード | 自動（CFシートのASIN行から） |
| 年月 | YYYY-MM | 自動 |
| 仕入単価（円） | 1個あたり原価 | 自動（CFシートの「仕入単価」行から） |
| 在庫数 | 月末在庫数 | 自動（CFシートの「在庫」行から） |

### データソース
- **キャッシュフロー管理シート**（既存のGoogle スプレッドシート）
- 社長が毎月1日に入力する棚卸しデータを活用
- シートURL は PropertiesService で管理（コードに直書きしない）

### 計算ロジック
- M1の「現在の仕入単価」= M2の該当ASINの最新月の単価
- D1の「仕入れ単価」= その販売日が属する月のM2単価を参照
- 将来改良: FIFO（先入先出法）で在庫ロット別の原価追跡
